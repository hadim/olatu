## 1. Owner feedback (verbatim intent, grouped)

Thirteen points, regrouped into four themes.

### Bugs (charts read wrong today)
- **B1 — x-axis years invisible.** uPlot's default time-axis drops the year at our
  zoom levels, so multi-year/All views show only month/day. Need an explicit tick
  formatter that always carries the year (and month context).
- **B2 — data points overflow their frame.** On short windows (e.g. recent data)
  point markers / cursor points spill past the panel edge into the title / next
  panel.
- **B3 — temperature panel hidden when empty.** Today the panel is *dropped* when
  sea-temp is sparse. The owner wants it **always present**, showing an explicit
  "data not available" state instead of vanishing (matches 0001 §6.2 / 0002 §4.7 —
  missing temp is a first-class UI state, never an empty/absent panel).
- **B4 — zoom has no delimiter.** Drag-to-zoom gives no visible selection
  rectangle, so you can't see what you're about to zoom into.

### Visual polish
- **P1 — type too small.** Increase font sizes across the board (labels are
  0.68–0.72rem today).
- **P2 — banner space usage.** The real-time section is fine in content but should
  use its space better — denser, more "pro instrument", genuinely beautiful.
- **P3 — staleness wording.** The word *"Vieillissant" / "Aging"* confused the owner.
  Keep **only a colour badge**, move it **top-right**, and reveal the explanation
  **on click** (popover) rather than spelling the state out inline.

### New interactions
- **N1 — hover value card.** On chart hover, show the **exact date + time** and the
  value of each series, in a **discreet panel** echoing the top banner (0001 §6.2
  "unified value card").
- **N2 — real time navigation.** The preset chips are too limited. Add a **custom
  date-range** picker plus an **elegant, discreet** way to jump to a specific **year**
  or a **month of a year** (0001 §6.3: per-year chips + cherry-picker + heat-ribbon).
- **N3 — smoothing control.** An intuitive toggle/slider to smooth the plotted
  curve. Decision: a small **segmented control (Raw / Light / Strong)** applying a
  centred rolling mean. Never smooth *across data gaps*, never smooth **direction**
  (circular — a rolling mean of degrees is meaningless).
- **N4 — directional spread plot.** Surface `peak_directional_spread_deg`.
  **Decision (owner): render it as a translucent ± band around the direction track**
  (0001 §6.2), not a separate panel.
- **N5 — mini-map.** Coordinates are fine, but add a small map of the buoy's
  position (0001 §6.4: static thumbnail + marker; lazy MapLibre on expand later).

### Content & docs
- **C1 — terminology (vague vs houle).** See decision below.
- **C2 — per-term definitions.** Every term gets a plain-language definition, one tap
  away (0001 §6.5 glossary). Includes spelling out that "significant" height is the
  **mean of the highest one-third (~top 33 %)** of waves — the "particular
  definition" the owner half-remembered (it is *not* a 30th percentile).
- **C3 — README.** Add an app screenshot to the header, refresh copy, and fix the
  **stale live URL** (the live site is now the custom domain **https://olatu.io**).

## 2. Decisions

### D1 — Terminology: "vague" for height, "houle" for direction/period
The buoy reports the **whole sea state** (wind-chop + swell combined), so:
- **Height** (`significant_wave_height_m`, `max_wave_height_m`) stays **"wave height /
  hauteur des vagues"** — technically correct, it is the combined sea state.
- **Peak direction & period** (`peak_direction_deg`, `peak_period_s` /
  `significant_period_s`) describe the **dominant** spectral component, which at 06403
  is usually the swell → labelled **"swell / houle"** ("swell direction",
  "direction de la houle", "swell period", "période de la houle").
- The **glossary explains the difference** (wave = full sea state; swell = the
  long-period organised part) so the choice is taught, not just asserted.

This is a labelling/i18n decision only; canonical schema names (0002) are unchanged.

### D1b — Banner layout: direction-only dial, wave height as the hero (right)
Follow-up owner feedback: the hero **Hs number inside the compass dial read as an
overlap** with the direction. Resolved by splitting the two:
- The **dial is direction-only** — arrow + a translucent **spread "cone"** (±spread°
  around the from-direction, i.e. how focused vs messy the swell is, the owner's
  "force") + N/E/S/W ticks + a centred compass/degrees readout. Caption shows
  "Direction de la houle · de NO · 312° · étalement ±23°".
- **Everything else moves to the right**, with **wave height as the dominant hero**
  (largest, accent-coloured), then Max wave / Swell period / Sea temperature as a
  supporting gauge row. Right-hand readouts are deliberately more prominent than the
  dial (per the owner: "un poil plus gros/présent par rapport à la direction").

### D2 — Directional spread = band around the direction track
Drawn as a translucent band spanning `dir ± spread` on the existing direction
panel. Reads as "clean vs confused" at a glance; keeps the panel count down.
Wrap-around at 0/360° is clamped for v1 (documented limitation; a proper cyclical
band can come with the direction-glyph layer in 0001 Phase 3).

## 3. Batches (delivery order)

**Batch 1 — fixes + polish + the two decisions** (this pass):
B1 x-axis years · B2 points overflow · B3 temp empty-state · B4 zoom delimiter ·
P1 fonts · P2/P3 banner rework + staleness badge · N1 hover value card ·
N3 smoothing · N4 spread band · C1 terminology · C2 *lite* (per-metric "i" info
popovers with simple definitions; the full slide-over glossary is Batch 2).

**Batch 2 — heavier features:**
N2 custom date-range + year/month navigation + heat-ribbon · N5 mini-map ·
C2 *full* 3-tier glossary slide-over (0001 §6.5) · C3 README + screenshot (done last,
so the screenshot shows the improved UI).

Batch order favours high-impact, low-risk changes first, and keeps the data layer
untouched (everything Batch 1 needs is already in `daily.parquet`'s HEADLINE columns,
incl. `peak_directional_spread_deg` and `significant_period_s`).

## 4. Implementation notes (Batch 1)

- **Data:** add `peak_directional_spread_deg` (+ `significant_period_s` for the swell-
  period label) to the webapp's `HISTORY_COLUMNS`; both already exist in
  `daily.parquet`. No ingest change.
- **Hover card:** keep uPlot `legend.show = false`; drive a custom DOM readout from a
  shared `setCursor` hook (idx → timestamp + per-series values), formatted via the
  existing `format.ts` helpers (locale + Europe/Paris).
- **Temp empty-state:** stop filtering the panel out; render it always and overlay a
  localized "data not available" message when coverage (`manifest.coverage`) is null
  / the loaded window has < N non-null points.
- **Zoom delimiter:** style uPlot's `.u-select` (accent-tinted fill + border) so the
  drag region is visible on both themes.
- **Points overflow:** clip series/point drawing to the plotting area + `overflow:
  hidden` on the panel + small marker radii.
- **Smoothing:** centred moving average over each non-direction series, gap-aware
  (reset the window across nulls). Direction/spread excluded.
- **Definitions:** a per-locale definitions map keyed by canonical variable name,
  surfaced through a reusable accessible `InfoPopover`. This is the seed of the
  Batch-2 glossary and the eventual Paraglide/JSON glossary in 0001 §8.
